Moono Protocol — это децентрализованный протокол кредитования на Solana, разработанный специально для запуска токенов на pump.fun.

## Приложение

Протокол задеплоен и доступен по адресам:

- **Основное приложение** — [app.moono.me](https://app.moono.me)
- **Зеркало в IPFS** — [IPNS ссылка](https://k51qzi5uqu5dkcd1cjf1nkei8zjdugoy2ynd7zyqfsb5bf0wmubjseuy98fw96.ipns.dweb.link)

Все релизы публикуются в IPFS. Если основное приложение по какой-то причине недоступно, вы всегда можете использовать протокол через зеркало в IPFS.

## Проблема

Запуск токена на pump.fun требует SOL для начальной ликвидности — нужно профинансировать кривую связывания (bonding curve) покупкой. Если у вас недостаточно SOL, запуск невозможен. Также существует риск блокировки собственного капитала в волатильном новом токене.

## Решение

Moono Protocol позволяет **занять SOL** для запуска и начального финансирования вашего токена на pump.fun, а **поставщики ликвидности получают проценты**, предоставляя SOL, который используют заёмщики.

### Для заёмщиков

1. Создайте многоразовый [**пресет запуска**](/ru/guides/launch-presets/) с метаданными токена, общим объёмом и количеством десятичных знаков — или сгенерируйте всё через встроенный AI-генератор
2. По желанию настройте [**bundle-кошельки**](/ru/guides/bundle-wallets/) — производные PDA-кошельки, которые распределяют начальную покупку между несколькими адресами в той же транзакции
3. Выберите сумму займа (0.1–10 SOL) и срок (1–24 часа)
4. Moono создаёт ваш токен на pump.fun, выполняет начальную покупку, делает bundle-покупки и опционально позволяет вам выкупить свою долю в той же атомарной транзакции
5. Купленные токены (начальная покупка + bundle-доли) служат залогом по займу
6. Погасите займ, чтобы вернуть токены, или самоликвидируйтесь, чтобы закрыть позицию в SOL

### Для поставщиков ликвидности

1. Внесите SOL в пул ликвидности протокола
2. Выберите **тик** (уровень риска), определяющий процентную ставку
3. Низкие тики = низкая ставка, но ваша ликвидность используется первой (выше утилизация)
4. Высокие тики = высокая ставка, используются только когда низкие тики исчерпаны
5. Проценты начисляются на позицию по модели кумулятивного индекса — видны как рост стоимости долей
6. **Заберите** накопленные проценты в любой момент или включите их в вывод/закрытие позиции

## Ключевые особенности

- **Запуск в одной транзакции** — займ, создание токена, начальная покупка, bundle-покупки и опциональная личная покупка пользователя выполняются атомарно
- **Пресеты запуска** — многоразовые шаблоны с произвольным mint, названием, символом, объёмом, десятичными знаками, URI метаданных и настройкой bundle-кошельков
- **Bundle-кошельки** — десятки производных PDA, которые делают дополнительные покупки в рамках того же запуска и распределяют первичный спрос по многим адресам
- **AI-метаданные** — сгенерируйте название, символ, описание и изображение токена в один клик; загрузка изображений в IPFS прямо из приложения
- **Многоуровневая ликвидность** — 1 024 уровня риска позволяют LP выбрать оптимальное соотношение риск/доходность
- **Кумулятивный индекс LP-процентов** — проценты точно учитываются по долям в момент выдачи займа и забираются отдельно от вывода
- **On-chain экономика** — все комиссии, проценты и логика ликвидации рассчитываются прозрачно on-chain
- **Самоликвидация** — заёмщики могут ликвидировать собственные займы (в любое время) и получить излишек средств; объединённое действие **Sell & Liquidate** также сворачивает позиции bundle-кошельков в одном потоке
- **Поддержка Jito** — опциональная отправка через Jito-bundles для лучшей доставки запусков и пользовательских покупок
- **Некастодиальный** — протокол представляет собой программу Solana; никто не может получить доступ к вашим средствам вне логики программы

## Поддерживаемые платформы

Moono Protocol спроектирован для поддержки **нескольких платформ запуска**. Первая интегрированная платформа:

- **pump.fun** — ведущий лаунчпад мемкоинов на Solana

В будущем будут добавлены новые платформы. Архитектура протокола нативно поддерживает несколько платформ запуска и котировочных активов — каждая платформа получает собственную конфигурацию с независимыми параметрами и собственным флагом `paused` для гранулярного контроля. Текущий деплой на mainnet настроен для pump.fun с WSOL (wrapped SOL) в качестве котировочного актива.
