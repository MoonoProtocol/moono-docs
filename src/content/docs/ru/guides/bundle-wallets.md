**Bundle** — набор кошельков под управлением протокола (PDA), которые делают дополнительные покупки в той же транзакции запуска, что и initial buy. Bundle распределяет давление покупки по многим кошелькам в одной атомарной операции, а затем позволяет заёмщику продать эти позиции и собрать SOL при сворачивании займа.

Bundle настраивается для каждого [пресета запуска](/ru/guides/launch-presets/) и неактивен, пока вы не запустите из пресета.

## Что такое bundle-кошельки

Каждый bundle-кошелёк — **Program Derived Address** (PDA), производный от:

- сида `"bundle"`
- адреса пресета
- индекса кошелька (0, 1, 2, …)

Поскольку адреса детерминированы, протокол подписывает за них в CPI без хранения приватных ключей. Отдельных кошельков, которыми вы управляете, нет — управляет протокол. Ваша задача — задать количество кошельков, профинансировать их SOL'ом для аренды и комиссий, а потом запустить sell/collect, когда сворачиваете позицию.

Количество bundle-кошельков задаётся в **Bundle Addresses Count** в пресете (максимум ограничен программой). Сами адреса хранятся в персональном для пресета **Address Lookup Table (Bundle ALT)**, чтобы транзакция запуска уложилась в лимиты Solana по аккаунтам.

## Зачем нужны bundle-кошельки

- **Распределённый initial fill** — вместо одного кошелька, который покупает всё, давление покупки расходится по N кошелькам в одной атомарной транзакции. Полезно, когда важно сразу иметь много холдеров после запуска.
- **Анти-снайп** — в связке с curve-checkpoint и поддержкой Jito bundle-покупки дают плотную атомарную последовательность, в которую снайперам сложно встроиться.
- **Сворачивание одним действием** — bundle-доли можно продать, а SOL собрать парой транзакций со страниц пресета и займа, без ручного отслеживания десятков кошельков.

Если эти свойства не нужны, можно запускать с `bundle_addresses_count = 0` (или вообще без пресета), и bundle-слой пропускается.

## Жизненный цикл

### 1. Создание Bundle ALT

На странице пресета (или при создании пресета) задайте **Bundle Addresses Count** и нажмите **Recreate ALT**. Приложение выводит все PDA bundle-кошельков и записывает их в новую Address Lookup Table, владельцем которой становится ваш кошелёк. Адрес ALT попадает в пресет.

Позже можно сделать **Amend ALT**, чтобы дополнить таблицу без полной пересборки — удобно при увеличении количества.

### 2. Финансирование bundle-кошельков

До первого запуска каждому bundle-кошельку нужны SOL для:

- Оплаты аренды per-launch token-ATA, которые он создаёт
- Покрытия комиссий за транзакции bundle sell / collect

На странице пресета есть таблица **Bundle Wallets** с индексом, адресом, ATA, volume accumulator и балансом SOL для каждого. Кнопка **Fund Wallets** пополняет их до рекомендованного уровня. Кошельки ниже порога подсвечены.

Транзакция финансирования отдельная и оплачивается с вашего кошелька.

### 3. Запуск

При запуске из пресета протокол по слайдеру **Initial Buy Amount** делит сумму займа: часть уходит execution-кошельку протокола на initial buy, остальное распределяется по bundle-кошелькам согласно **Bundle Distribution Type** (сейчас **Equal**). Bundle-покупки происходят в той же атомарной транзакции (или Jito-bundle), что и initial buy.

Объём base-токенов на bundle-кошельках учитывается отдельно в `loan.bundled_base_amount`, не смешиваясь с залогом initial buy.

### 4. Продажа и сбор SOL

Для сворачивания есть три пути:

- **Sell & Liquidate** (страница займа) — один клик; продаёт все base-балансы bundle-кошельков, затем ликвидирует займ
- **Sell Selected** (пресет → Bundle Wallets) — выбираете конкретные bundle-кошельки и продаёте их base через `bundle_sell_all_market_0`
- **Collect SOL** (пресет → Bundle Wallets) — собирает остатки SOL с bundle-кошельков обратно вам через `bundle_collect_all`

Действия sell/collect принимают произвольные подмножества индексов, так что операцию можно разбить на несколько транзакций (Solana ограничивает кол-во аккаунтов на tx).

## Отображение SPL

Таблица Bundle Wallets умеет показывать баланс SPL по любому mint (по умолчанию — base-mint запуска). Кнопка **SPL token** над таблицей переключает отображаемый mint — полезно при проверке предзапускового финансирования или балансов другого токена после запуска.

## Привязка займа со стороны пресета

Если вы возвращаетесь к пресету позже без активной ссылки на займ, но на bundle-кошельках всё ещё лежат токены, нажмите **Try to Find Loan** — приложение найдёт недавний займ, использовавший этот пресет, и подставит его адрес для маршрутизации sell/collect.

## Типы распределения

| Тип | Поведение |
|---|---|
| **Equal** (0) | Протокол делит bundle-долю суммы займа на N равных частей, по одной на bundle-кошелёк |

В будущем могут появиться другие типы распределения.

## Советы

- **Recreate ALT после изменения количества** — ALT должна содержать ровно те PDA bundle-кошельков, которые ожидает программа
- **Финансируйте перед каждым запуском** — фонды переживают запуски, но расходуются на комиссии sell/collect и аренду ATA
- **Используйте Jito для атомарности** — без Jito bundle и ваша user buy могут попасть в разные слоты
- **Сначала продайте, потом собирайте** — bundle-кошельки сначала должны конвертировать base-токены обратно в SOL, прежде чем будет что собирать; **Sell Selected** делает продажу, **Collect SOL** забирает выручку
